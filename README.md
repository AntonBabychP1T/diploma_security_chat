# Secure LLM Chat & Assistant

Цей проект — це комплексна система для безпечного спілкування з великими мовними моделями (LLM) та використання AI-асистента ("Секретаря"). Головною особливістю є захист персональних даних (PII) та інтеграція з робочими інструментами (пошта, календар).

## 🚀 Основні можливості

*   **Безпечний чат (PII Masking):** Система автоматично виявляє та маскує чутливі дані (email, телефони, номери карток) перед відправкою запиту до LLM. Відповідь від моделі "розмаскується", зберігаючи контекст, але не передаючи реальні дані провайдеру AI.
*   **AI Секретар (Agentic Mode):** Розумний помічник, який може керувати вашою поштою та календарем.
    *   Інтеграція з **Google Workspace** (Gmail, Calendar).
    *   Інтеграція з **Microsoft 365** (Outlook, Calendar).
    *   Можливість читати пошту, створювати події, відповідати на листи.
*   **Мультимодельність:** Підтримка різних провайдерів AI:
    *   OpenAI (GPT-5.5, GPT-5.4, GPT-5.4 mini/nano)
    *   Google Gemini
*   **Довготривала пам'ять:** Агент запам'ятовує контекст розмов та інформацію про користувача для більш персоналізованих відповідей.
*   **Аутентифікація:**
    *   Локальна реєстрація (Email/Password).
    *   OAuth через Google та Microsoft.
*   **Аналітика та Адміністрування:**
    *   Дашборд для перегляду метрик використання токенів, активності користувачів.
    *   Управління чатами та історією.
*   **Голосове управління:** Підтримка транскрипції аудіо повідомлень.

## 🛠 Технологічний стек

### Backend (Серверна частина)
*   **Мова:** Python 3.x
*   **Фреймворк:** FastAPI (асинхронний, високопродуктивний веб-фреймворк).
*   **База даних:** SQLite (з асинхронним доступом через `aiosqlite`).
*   **ORM:** SQLAlchemy 2.0.
*   **AI Інтеграції:** OpenAi API, Google Generative AI SDK.
*   **Безпека:** `python-jose` (JWT), `passlib` (хешування паролів), кастомний модуль PII Masking.

### Frontend (Клієнтська частина)
*   **Фреймворк:** React (Vite).
*   **Мова:** TypeScript.
*   **Стилізація:** Tailwind CSS.
*   **UI Компоненти:** Lucide React (іконки), Recharts (графіки).
*   **Стейт-менеджмент:** React Context / Hooks.

## ⚙️ Інструкція по запуску

Для запуску вам знадобиться встановлений **Python** (версії 3.10+ рекомендовано) та **Node.js** (версії 18+).

### 1. Налаштування Backend

1.  Перейдіть у папку `backend`:
    ```bash
    cd backend
    ```

2.  Створіть віртуальне середовище (рекомендовано):
    ```bash
    python -m venv venv
    # Windows
    venv\Scripts\activate
    # macOS/Linux
    source venv/bin/activate
    ```

3.  Встановіть залежності:
    ```bash
    pip install -r requirements.txt
    ```

4.  Налаштуйте змінні середовища:
    *   Скопіюйте файл `.env.example` у файл `.env`.
    *   Відкрийте `.env` та заповніть необхідні ключі (API Keys для OpenAI/Gemini, налаштування БД тощо).
    ```bash
    cp .env.example .env
    ```
    *Приклад вмісту .env:*
    ```env
    OPENAI_API_KEY=sk-....
    GEMINI_API_KEY=...
    DATABASE_URL=sqlite+aiosqlite:///./chat.db
    # ... інші налаштування
    ```

5.  Запустіть сервер:
    ```bash
    uvicorn app.main:app --reload
    ```
    Сервер буде доступний за адресою: `http://localhost:8000`.
    Документація API (Swagger UI): `http://localhost:8000/docs`.

### 2. Налаштування Frontend

1.  Відкрийте новий термінал та перейдіть у папку `frontend`:
    ```bash
    cd frontend
    ```

2.  Встановіть залежності:
    ```bash
    npm install
    ```

3.  Налаштуйте змінні середовища (якщо необхідно):
    *   Зазвичай Vite автоматично підтягує `.env` файли, переконайтеся, що API URL вказано вірно (за замовчуванням може бути налаштовано на `http://localhost:8000`).

4.  Запустіть режим розробки:
    ```bash
    npm run dev
    ```
    Застосунок буде доступний за адресою, яку вкаже Vite (зазвичай `http://localhost:5173`).

## 🧠 Як це працює (Архітектура)

1.  **Запит користувача:** Користувач відправляє повідомлення через React-інтерфейс.
2.  **PII Сервіс:** На бекенді повідомлення проходить через `PIIService`. Регулярні вирази та логіка визначають чутливі дані (наприклад, `test@email.com`) і замінюють їх на плейсхолдери (наприклад, `<EMAIL_1>`).
3.  **LLM Обробка:** "Очищений" текст відправляється до обраної моделі (GPT-5.5/GPT-5.4, Gemini). Модель генерує відповідь, використовуючи плейсхолдери.
4.  **Відновлення (Unmasking):** Отримана відповідь проходить зворотну обробку, де плейсхолдери замінюються на оригінальні дані, якщо це необхідно (або залишаються, якщо контекст дозволяє).
5.  **Агент (Секретар):** Якщо активовано режим агента, запит аналізується на наявність команд (наприклад, "створи зустріч"). Якщо намір розпізнано, викликаються відповідні сервіси (`GoogleWorkspace`, `MicrosoftGraph`) для виконання дії через API провайдера.
6.  **Збереження:** Історія чату та вектори пам'яті зберігаються в базі даних SQLite.
